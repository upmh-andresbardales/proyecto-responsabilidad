\section{Resumen Ejecutivo}
Este proyecto implementa una plataforma IoT para monitoreo acuaponico en tiempo real. La solucion integra simulacion de sensores, mensajeria MQTT, persistencia en MongoDB, API REST en FastAPI y visualizacion web en Vue 3.

La arquitectura fue disenada para ser agnostica a la fuente de datos. Actualmente opera con datos sinteticos, pero esta lista para integrar sensores fisicos mediante ESP32 o Raspberry Pi sin cambios mayores en la logica de negocio.

El sistema ya cumple el flujo completo de extremo a extremo:
\begin{enumerate}
\item Generacion de datos de sensores.
\item Publicacion por MQTT en EMQX.
\item Consumo en backend y guardado en MongoDB.
\item Generacion de alertas por umbrales.
\item Exposicion de datos por API REST.
\item Visualizacion en dashboard web en tiempo real.
\end{enumerate}

\newpage
\section{Arquitectura General del Sistema}
\subsection{Componentes Principales}
\begin{itemize}
\item \textbf{EMQX}: broker MQTT para mensajeria IoT.
\item \textbf{Simulator}: publica datos sinteticos cada 5 segundos.
\item \textbf{Backend FastAPI}: suscribe MQTT, persiste datos y expone API.
\item \textbf{MongoDB}: almacena lecturas, alertas y usuarios.
\item \textbf{Frontend Vue 3}: dashboard con graficas, alertas y autenticacion.
\end{itemize}

\subsection{Diagrama de Flujo Logico}
\begin{enumerate}
\item El simulador publica mensajes en topicos MQTT de sensores.
\item EMQX distribuye mensajes a suscriptores.
\item El backend consume mensajes, valida y almacena en MongoDB.
\item Si se detectan umbrales fuera de rango, el backend crea alertas.
\item El frontend consulta datos por REST y escucha cambios en tiempo real por MQTT WebSocket.
\end{enumerate}

\section{Backend FastAPI}
\subsection{Responsabilidades}
\begin{itemize}
\item Exponer API REST para sensores, alertas, usuarios y autenticacion.
\item Suscribirse a MQTT para recibir telemetria.

\section{Conclusiones}
El proyecto se encuentra funcional en su arquitectura digital completa. La cadena de monitoreo, almacenamiento, visualizacion y gestion de usuarios esta implementada y probada. El sistema puede operar hoy con datos sinteticos y esta preparado para transicionar a hardware real.

En terminos de entrega academica, el avance actual presenta evidencia solida, trazable y alineada con los objetivos definidos para la plataforma acuaponica inteligente.
